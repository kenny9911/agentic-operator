# QA Test Report — Import Manifest + Tenant Flows
Date: 2026-05-20 · QA: test-team agent

## TL;DR

- **Backend is ready to ship.** 76/76 manifest-import tests pass, 25/25 regression sanity tests pass, all three workspaces typecheck clean, every HTTP probe behaves correctly. The contract from `docs/design/import-workflow-manifest.md` is implemented end-to-end.
- **Production (App Router) frontend is mocked.** `ImportManifestModal.tsx:130-137` is `setTimeout(900ms)` against a static sample; `+ New tenant` in `tenant-switcher.tsx:167-174` just toasts "Not yet implemented"; `chrome.tsx:63` reads tenants from a static fixture; `DataProvider` useEffect has `[]` deps (`data-context.tsx:143`) so the canvas never re-fetches on tenant change.
- **Legacy SPA (the actually-served `/portal/index.html`) is wired for Import, but `+ New tenant` is a no-op button.** `views/import-manifest.jsx` makes three real `/v1/tenants/:slug/manifest-import*` fetches. But `app.jsx:253-255` renders the "New tenant" button with no `onClick`, and `views/tenants.jsx` (which DOES have a real CreateModal) is not loaded by `index.html`.
- **Data residue is severe.** 290 tenants in SQLite; only 4 look real. 286 are test residue.
- **Verdict: HOLD on production App Router. Ship-ready on backend + legacy SPA Import.** 4 blockers below; ~1 day to wire the production modals to the existing (already-correct) backend.

## Blocker findings

1. **[BLOCKER · UX]** Production `+ New tenant` is a stub toast (`tenant-switcher.tsx:167-174`). A fully implemented `TenantCreateModal` exists (`TenantCreateModal.tsx`, 612 lines, POSTs to `/v1/tenants`) but is never imported. No way to create a tenant from the actively-served UI surface, even though the backend works. **~30 LoC fix:** import `TenantCreateModal`, replace toast with `setShowModal(true)`, on success refresh tenant list + `useTenantNavigate(newTenant)`.

2. **[BLOCKER · Feature]** Production `ImportManifestModal` is mocked (`ImportManifestModal.tsx:130-137`). The 1132-line modal contains zero `fetch()` calls. `views/import-manifest.jsx` (legacy SPA) already shows the wiring: `/v1/tenants/:slug/manifest-import/fetch-url`, `…/manifest-import` validate + commit. **~150 LoC fix:** port the three legacy fetch helpers, swap `buildSampleParse()` for the real `ManifestImportPreview` shape, hook up the existing `OverwriteConfirmModal.tsx` to the 409 path.

3. **[BLOCKER · Data]** `DataProvider` does not re-fetch on tenant change (`data-context.tsx:143` `[]` deps). Even after fixing the modals, switching tenants shows the previous tenant's data. **~10 LoC fix:** read `useTenant()` and add it to the deps; `app/api/spa/bootstrap/route.ts:18-23` also needs to accept `?tenant=…` and forward it to `/v1/*` so the api scopes per request. Alternative: drop the proxy and call `/v1/*` directly with `credentials: same-origin` — auth plugin already supports cookie-based tenants.

4. **[BLOCKER · Hygiene]** 286 test-residue tenants in SQLite. Count grew by 30 during this audit while concurrent test suites ran. Risk: production switcher (once wired) is unusable. **~20 LoC fix:** vitest `afterAll` that deletes by known prefixes (`budget-%`, `evt-tester-%`, `webhook-tenant-%`, `mem-test-%`, `aud-other-%`, `qa-probe-%`), or standardize on a `test-tenant-*` prefix.

5. **[MAJOR · UX]** Legacy SPA `+ New tenant` button (`app.jsx:253-255`) has no `onClick`. The legacy SPA IS the live UI (`next.config.mjs:30`), so this is the user-facing blocker today. **Fix:** load `views/tenants.jsx` from `index.html`, mount it as a view in `app.jsx`, wire the button. The wiring already exists in `tenants.jsx`.

6. **[MAJOR · UX]** Legacy SPA does not refresh data on tenant switch (`app.jsx:237` `setTweak("tenant", t.id)` is purely UI). `data.js:63-81` exposes `fetchBootstrap()` already — needs the tenant setter to invoke it and apply the new payload.

7. **[MINOR · Doc]** `CLAUDE.md:74-77` says legacy SPA is the active UI and App Router portal pages are dead, but `apps/web/app/portal/[tenant]/(views)/workflows/page.tsx` is outside the `_portal_legacy/` guard and Next compiles it. The doc and file layout disagree. Resolve by deleting one tree or updating CLAUDE.md.

8. **[MINOR · Env]** Node version mismatch produced 5 false-positive suite failures (`ERR_DLOPEN_FAILED`) on the first run before `nvm use 26`. CLAUDE.md documents this, but a `predev`/`pretest` Node-version assert would save the next contributor 10 minutes.

1. **Wire production `TenantCreateModal`.** Import it from `tenant-switcher.tsx`, gate behind `useState` open flag, replace the stub toast at 167-174 with `setShowCreate(true)`. On success: refresh tenant list + `useTenantNavigate(created.slug)`. With `mintToken:true`, also surface `TenantTokenRevealModal.tsx`.

4. **Replace `chrome.tsx`'s static `TENANTS` import with a `useTenants()` hook.** SWR/TanStack query against `/v1/tenants?include_inactive=false`; freshly created tenants appear without a refresh.

5. **Wire legacy SPA `+ New tenant`.** Add `<script src="/portal/views/tenants.jsx">` to `index.html`, mount the view in `app.jsx`'s router, wire the button to `navigate("tenants")`. The wiring already exists inside `tenants.jsx`. (Or delete the button if legacy SPA is being retired.)

6. **Vitest `afterAll` cleanup for test tenants.** Standardize on a `test-tenant-*` / `qa-*` prefix and tear down by prefix. Drains the 286 residue rows on next test run.

7. **`predev`/`pretest` Node-version check.** `node --version | grep -q v26` with a friendly error pointing at `nvm use`.

8. **Resolve App-Router-vs-SPA duplication.** Two copies of every feature exist (legacy = real backend; App Router = setTimeout mock). Pick one — this is the single biggest source of QA confusion in the repo.

## Appendix: backend verdict

The Import-Workflow-Manifest backend is the cleanest implementation in the repo right now. Every audit finding from `import-workflow-manifest-review.md` (4 blockers + 8 majors) has a corresponding passing test — SSRF guard across RFC1918/CGNAT/IPv6 ULA/AWS metadata (35 tests), compound overwrite rule (18 tests covering priorN=1/3/10/100), `dpl-` session ID convention, `audit_log` routing (no `events`-table pollution), and the disk→tx→rename ordering implied by the commit phase tests. When the frontend modals get rewired, the backend will not be the bottleneck.
